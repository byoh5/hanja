# HanjaStep 웹 기반 한자 급수 학습 앱(PWA) 사업/서비스 기획서 (실무 초안)

- 문서 버전: v0.9
- 작성일: 2026-02-20
- 기준 기관: 한국어문회(본 문서는 요청안의 급수/배정자 수 기준으로 설계)
- 문서 목적: 서비스 기획, 개발 착수, MVP 검증, 확장 로드맵 의사결정

---

## 1. 서비스 개요

### 1.1 서비스명(가칭)
- HanjaStep

### 1.2 서비스 목적
- 한자 급수 시험 준비를 위한 웹 기반 학습 플랫폼 제공
- 급수별 배정 한자를 체계적으로 학습/복습할 수 있는 루프 설계
- 모바일/PC 어디서나 학습 가능하며, 오프라인 환경에서도 핵심 기능 사용 가능(PWA)

### 1.3 핵심 가치 제안(Value Proposition)
- 기관 기준이 명확한 급수 학습: 한국어문회 기준 고정
- 단순 암기 앱이 아닌 자동 복습 엔진(SRS) 중심 학습
- 학습-시험-오답분석을 하나의 플로우로 통합

---

## 2. 시장 배경 및 문제 정의

### 2.1 현재 시장 문제
- 단발성 카드 암기 중심으로 장기 기억 전환이 약함
- 급수 체계가 기관 혼합되어 사용자 혼란 발생
- 시험 기능이 있더라도 오답 복습으로 연결되지 않음
- 오프라인 지원 부족으로 통학/이동 시간 학습 제약 존재

### 2.2 해결 전략
- 기관 기준 단일화(한국어문회)
- 급수별 배정 한자 데이터 고정 및 버전 관리
- SRS 기반 자동 복습 큐 도입(오답 우선, 간격 증가)
- 실전 시험 모드와 약점 분석을 학습 플로우에 결합
- PWA 기반 오프라인 지원

## 3. 타겟 사용자 및 페르소나

### 3.1 1차 타겟
- 초등학생(8급~6급) + 학부모(학습 관리자)

### 3.2 2차 타겟
- 중학생 이상 급수 준비생
- 공무원/기업 가산점 준비자

### 3.3 대표 페르소나
1. 초등 4학년 수험생 + 보호자
- 니즈: 짧은 시간 반복 학습, 시험 전 점검
- 페인포인트: 아이의 진도/취약 한자 확인 어려움

2. 직장인 재수험생
- 니즈: 출퇴근 오프라인 학습, 약점 위주 복습
- 페인포인트: 학습 시간 분절, 복습 타이밍 관리 어려움

### 4.3 북극성 지표(NSM)
- `주간 마스터(Review/Mastered) 한자 수`

## 7. 기능 상세 명세

## 7.1 급수 선택
- 사용자 스토리: “나는 목표 급수를 선택하고 해당 범위만 집중 학습하고 싶다.”
- 주요 기능
  - 목표 급수 선택
  - 선택 급수 기준 학습 진도판 표시
  - 완료율/남은 학습량 표시
- 수용 기준(AC)
  - 급수 선택 후 1초 내 학습 목록 렌더링
  - 앱 재실행 시 마지막 목표 급수 유지

## 7.2 한자 학습 카드
- 카드 구성
  - 한자(대형)
  - 음/훈
  - 예시 단어 1~2개
  - TTS
- 상호작용
  - `알겠어요`: 난이도 하향, 다음 복습 간격 증가
  - `다시보기`: 오답 큐 우선 배치
- AC
  - 카드 전환 지연 100ms 이하 체감
  - TTS 실패 시 텍스트 fallback 제공

## 7.3 시험 모드
- 유형
  - 뜻 맞추기
  - 음 맞추기
  - 한자 선택
  - 혼합 랜덤
- 옵션
  - 문제 수(10/20/50)
  - 제한 시간 on/off
  - 오답만 재시험
- 결과
  - 점수, 소요시간, 유형별 정답률
  - 취약 한자 자동 복습 큐 등록

## 7.4 복습 알고리즘(SRS)
- 상태
  - `NEW` -> `LEARNING` -> `REVIEW` -> `MASTERED`
- 규칙(단순화)
  - 정답 시: `streak +1`, `interval` 증가(1, 2, 4, 7, 14, 30일)
  - 오답 시: `streak = 0`, `interval = 1`, `wrongCount +1`, 상태 `LEARNING`
  - `wrongCount` 상위 항목은 당일 세션 우선 출제
- 일일 큐
  - `dueDate <= today` 우선
  - 신규 학습량 일일 상한(예: 무료 20자)

## 7.5 급수 조회 기능
- 입력: 한자 1글자
- 출력
  - 해당 글자 최초 등장 급수
  - 상위 급수 포함 여부
- 예외 처리
  - 미등록 글자: “데이터셋 외 한자” 안내
- 확장
  - 단어 입력 시 구성 한자의 최고 난이도 급수 계산

## 7.6 학부모/관리자 리포트(Phase 3)
- 주간 학습일수, 누적 학습량, 오답 TOP10
- 시험 모드 점수 추이
- PDF 다운로드

## 8. UX/화면 구조(정보 아키텍처)

### 8.1 화면 목록
1. 온보딩/급수 선택
2. 대시보드(오늘 할 일, 진도, 약점)
3. 학습 카드
4. 시험 모드 설정/진행/결과
5. 오답 노트
6. 급수 조회
7. 설정(음성/TTS/알림/데이터 백업)

### 8.2 핵심 사용자 플로우
1. 첫 방문: 급수 선택 -> 진단 퀴즈(선택) -> 오늘 학습 시작
2. 일일 학습: `오늘의 복습` -> `신규 학습` -> `짧은 시험`
3. 시험 대비: 혼합 모드 3회 -> 오답만 재시험 -> 약점 카드 집중

## 9. 기술 아키텍처

### 9.1 프론트엔드
- React + Vite + TypeScript
- UI: TailwindCSS
- 상태관리: Zustand(로컬 상태) + Dexie(영속)

### 9.2 데이터 저장
- IndexedDB(Dexie)
- JSON 기반 급수/메타 데이터 로딩

### 9.3 PWA/오프라인
- Service Worker(Workbox 권장)
- 캐시 전략
  - 앱 셸: `Cache First`
  - 콘텐츠 JSON: `Stale While Revalidate`
  - 분석/원격 동기화: `Network First`

### 9.4 배포
- GitHub Pages 또는 Cloudflare Pages
- CI: main 머지 시 빌드/배포 자동화

### 9.5 관측(Analytics)
- 이벤트 추적
  - `grade_selected`
  - `card_known`
  - `card_retry`
  - `quiz_completed`
  - `review_completed`
