# HanjaStep MVP 백로그 (개발 티켓 단위)

- 문서 버전: v0.1
- 작성일: 2026-02-20
- 기준 문서: `docs/hanjastep-prd.md`
- 대상 범위: Phase 1 (2~3주, 8급 MVP)

## 1. 목표와 범위

### 1.1 이번 백로그의 목표
- 8급 한자 학습/복습/시험의 핵심 루프를 웹앱에서 완성
- 오프라인 환경에서도 핵심 학습 플로우 동작
- 공개 베타 배포 가능한 품질 확보

### 1.2 포함(In Scope)
- 온보딩/급수 선택(8급 고정 가능)
- 학습 카드(한자/음/훈/예시/TTS)
- SRS 복습 큐(NEW, LEARNING, REVIEW, MASTERED)
- 시험 모드(뜻, 음, 한자 선택, 혼합)
- 결과/약점 반영, 오답 재시험
- PWA 설치 및 오프라인 캐시
- 기본 이벤트 트래킹, CI/CD

### 1.3 제외(Out of Scope)
- 7급 이상 콘텐츠
- 결제/구독
- 계정 로그인 및 클라우드 동기화
- 학부모 리포트/PDF 출력

## 2. 운영 기준

### 2.1 우선순위 정의
- `P0`: MVP 출시 필수
- `P1`: 출시 전 권장
- `P2`: 출시 후 개선

### 2.2 상태 정의
- `TODO`: 미착수
- `IN_PROGRESS`: 진행 중
- `BLOCKED`: 외부 이슈로 중단
- `REVIEW`: PR/QA 검토 중
- `DONE`: 완료

### 2.3 Definition of Ready (DoR)
- 요구사항/수용기준(AC)이 명시됨
- 의존성 및 선행 티켓 확인됨
- 테스트 방법(수동/자동)이 정의됨

### 2.4 Definition of Done (DoD)
- AC 충족
- 테스트 통과(관련 유닛/통합/E2E)
- 접근성 기본 점검(키보드, ARIA 라벨)
- 문서/릴리즈 노트 반영

## 5. 상세 티켓 명세

## MVP-001 React/Vite/Tailwind/PWA 초기 세팅
- 유형: Tech Setup
- 우선순위: P0
- 설명: TypeScript 기반 Vite 프로젝트에 Tailwind와 PWA 플러그인을 구성한다.
- AC
  - `npm run dev`, `npm run build`, `npm run preview` 정상 동작
  - 기본 `manifest.webmanifest` 생성
  - 모바일 viewport 및 앱 아이콘 기본 설정
- 산출물
  - 프로젝트 초기 디렉토리 구조
  - 환경 변수 예시 파일(`.env.example`)
